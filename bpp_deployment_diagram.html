package Deployment [ Planned production deployment on Render ]
Deployment analysis summary. The diagram shows the planned production model found in the repository: a single Render Docker Web Service that serves the React SPA build from Laravel public through nginx and forwards Laravel API traffic to PHP-FPM. The concrete codebase elements are the Laravel entrypoint backend/public/index.php, React entrypoint frontend/src/main.jsx, Dockerfile, render.yaml, docker/start.sh, nginx template, Sanctum API routes, PostgreSQL configuration, Meteo.lt and Perenual service configuration, local Laravel storage, file cache/session settings, sync queue, and SMTP/log mailer configuration. Render hosting, managed PostgreSQL, Render HTTPS termination, and production environment variables are planned hosting elements from the deployment files. The previous local workstation, Vite development server, Laravel artisan server, localhost API proxy, and local database host were removed from the runtime view because they describe development execution rather than final deployment.
